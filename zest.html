Zest Iterative Redesign

Team

Tiffany Huang
Ryan Peng
Cynthia Wang


Context

UI/UX class @ Brown University


Duration

3 weeks, ~8 hours/week


Tools

Figma

In this project, our team of designers worked with Zest, a B2B corporate gifting platform, to develop interface changes to accomodate a new shipping option.

The Problem: Lack of support for bulk ordering

While users of Zest's gifting platform were able to choose between shipping gifts to individual addresses and sending as e-gifts, there was no option to deliver all gifts to one address. Users would have to call customer service in order to do so.

Reimagining shipping methods

Each team member brainstormed two end-to-end flows for possible implementations of a bulk shipping process. While ideating, many key questions arose:

  • Are per-recipient UI elements (e.g. personalized gift messages) necessary since users who ship to one address are likely bulk ordering (e.g. for 100+ people)?
  • How do we want to enable bulk shipping? Some ideas included a drop-down menu, a simple checkbox, and a third box in the delivery method section.
  • Will adding a shopping cart enhance user experience? How will it interact with the rest of the user flow?

After sketching, we worked synchronously to create a lo-fi wireframe. Our key design decisions were as follows:

  • Added new delivery method under existing "ship to receipients' addresses" option
  • Chose to use a dropdown over a checkbox--we wanted it to be of equal importance to shipping to individual addresses
  • Added an order summary and shopping cart to clearly communicate shipping costs and reduce the need to click back and forth between the main page and checkout
  • Removed addresses from recipients list, as bulk shipping uses a single address
